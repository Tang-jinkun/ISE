# Second DOCX Generalization Proof Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prove that a second, non-Indo-Pak DOCX with only grounded start/end coordinates can produce a playable scene containing an AWACS, two exact fighter formations, one geometrically confirmed destruction, and one unresolved engagement without scenario-specific compiler branches or external trajectory JSON.

**Architecture:** Add evidence-backed `EngagementIntent` records to `SceneBlueprint`, resolve launcher/weapon/target actor groups before choreography, and make `ChoreographyCompiler` consume those relations instead of country and aircraft-name conditionals. Reuse the existing deterministic start/end trajectory synthesizer and actual GLB registry, then validate the complete Dockerized Agent export at artifact level.

**Tech Stack:** TypeScript 5.9, Zod, Node test runner via `tsx`, Mammoth DOCX parsing, Python `python-docx`, PowerShell, Docker Compose, existing ISE runtime contracts and Preview UI.

## Global Constraints

- The acceptance input starts from a real DOCX and does not include or attach any trajectory JSON.
- The new document must select `generic/v1`; do not add a North Sea scenario pack or any scenario-ID branch.
- Explicit quantities are authoritative: one AWACS, four Rafale, four J-10, and one missile per launch.
- Interaction correctness has priority over subtitle duration. Subtitles define narrative windows, but terminal interaction geometry may extend beyond them.
- A claimed destruction becomes a resolved runtime interaction only when weapon and target geometry meet; otherwise emit `unresolved`.
- An explicitly unconfirmed engagement remains unresolved and must not receive a fabricated interaction point or destruction state.
- Reuse existing E-3A, Rafale, J-10, and PL-15E GLBs. Do not download or commit new models for this proof.
- Runtime remains unaware of DOCX parsing, ScenarioPack selection, and generated-versus-catalog trajectory provenance.
- Validate only focused tests, one real Docker export, and user manual desktop Preview. Do not run mobile or broad Playwright suites.
- Do not write model credentials to source, environment files, logs, commits, exported artifacts, or replies. Inject credentials only into the Agent process environment.
- Do not fix unrelated legacy fixtures, full typecheck debt, visual polish, or non-blocking regressions in this milestone.
