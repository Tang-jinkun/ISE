# 新版脚本对话与场景工作台 UI 设计

## 背景

当前新版脚本页面采用固定双栏布局：左侧是智能问答，右侧是需要手动打开的“解析结果”。这套结构存在四个问题：

- 尚无产物时，右侧长期空置，压缩了主要对话空间；
- “解析结果”无法表达 EventPlan、字幕、场景蓝图、资源解析和运行时预览之间的阶段差异；
- 对话、审核、产物查看和参数编辑混杂在一个过大的页面组件中；
- 页面仍围绕 NarrativePlan v1 组织，而主任务已经确定向 NarrationPlan、SceneBlueprint 和 SceneProjectConfig v2 演进。

新版 UI 采用“对话主界面 + 自适应场景工作台”。对话区负责表达意图、附件发送、Agent 执行过程和最终答复；场景工作台负责审核、检查、编辑和预览阶段化产物。

## 目标

1. 没有可检查产物时，对话区使用全部可用宽度。
2. DOCX 与纯文本都可以独立启动场景生成，不要求必须上传文件。
3. 文件选择只创建待发送附件，点击发送后才上传并创建或复用 Agent Session。
4. EventPlan 出现后自动打开场景工作台，并把审核作为当前主要操作。
5. 字幕在场景生成前成为独立、可见、可编辑的权威产物。
6. 图片、视频、模型、航迹和相机按字幕或 Scene Beat 聚合检查。
7. 失败状态显示准确阶段、具体诊断和可执行的恢复操作。
8. 新版作为默认入口，旧版保留明确的回退按钮。
9. 提供云端和本地 OpenAI-compatible 模型配置入口，API key 只保存在 Agent 进程内存中。
10. UI 外壳可以先于新运行时契约实现，产物页签通过阶段映射适配 v1 和 v2。

## 非目标

- 本阶段不重新设计旧版脚本编辑器。
- 不在浏览器存储、数据库、环境文件或日志中持久化模型 API key。
- 不在前端实现 EventPlan、字幕、蓝图或运行时编译逻辑。
- 不新增移动端专项开发和验收；保留现有响应式行为，最终仅验收桌面端。
- 不把原始 JSON 作为普通用户的主要编辑界面。

## 方案选择

### 采用：自适应双区工作台

页面默认只有对话区。当出现 EventPlan 或更下游的产物时，右侧场景工作台自动展开。工作台可折叠和调整宽度，但不会在无内容时显示空白占位。

该方案保留对话的连续性，又为审核和编辑产物提供稳定区域，适合当前阶段化生成流程。

### 未采用：固定双栏

固定双栏实现简单，但无产物时浪费空间，并继续强化“所有解析结果都是同一种内容”的错误心智模型。

### 未采用：IDE 式自由停靠面板

自由停靠面板扩展性强，但需要额外的布局持久化、拖放和面板生命周期管理，不符合当前纵向链路优先级。

## 页面布局

### 顶栏

顶栏保持紧凑，包含：

- 返回项目列表；
- 可编辑项目标题；
- 当前模型提供商与模型的状态入口；
- 产物导出菜单；
- 保存；
- 场景预览；
- 返回旧版。

模型入口使用提供商图标或通用模型图标加简短模型名。未配置时显示“配置模型”，配置异常时显示警告状态。

### 对话区

对话区是页面的主要操作面，包含：

- 用户消息及其附件；
- Agent Turn 的思考摘要、工具活动、审核等待和最终答复；
- 准确的 queued、running、awaiting_review、completed、failed、cancelled 状态；
- 当前审核卡片；
- 固定在底部的消息编辑器。

执行过程默认在运行时展开，成功后折叠，失败时保持展开。失败 Turn 的标题显示“执行失败”，不能使用“已完成 N 步”。

移除长期占位的“生成大纲”和“细化场景”大按钮。常用动作由工作台当前阶段的主按钮承载；聊天输入仍支持自然语言命令。

### 消息编辑器

编辑器遵循 Codex 风格的显式发送交互：

- 附件按钮位于编辑器工具栏；
- 选择文件后，在输入框内显示附件标签、文件名、大小和移除按钮；
- 选择、替换或移除附件不会产生网络请求；
- 文本或附件任一存在即可发送；
- 纯文本、纯 DOCX、文本加 DOCX 都是有效请求；
- 点击发送后依次执行上传、创建或复用 Session、挂载附件和发送消息；
- 发送成功后清空输入和附件；失败时完整保留，允许重试；
- `Enter` 发送，`Shift+Enter` 换行；发送中禁用重复提交。

### 场景工作台

场景工作台由产物状态驱动，不由用户点击“开始解析”驱动。

- 无产物：工作台不渲染，对话区全宽；
- EventPlan 草稿：自动打开并进入“事件计划”；
- EventPlan 已批准：开放“字幕脚本”；
- NarrationPlan 就绪：开放“场景蓝图”，并提供“按字幕生成场景”；
- SceneBlueprint 或 ResolvedScenePlan 就绪：开放“资源”和“参数”；
- SceneProjectConfig 就绪：开放“预览”；
- 下游失败：保留最近成功产物，自动定位到失败阶段。

工作台桌面端默认占页面宽度的 42%，允许在 34% 到 58% 之间拖动。折叠后对话区占满剩余宽度。宽度仅保存在当前页面会话中。

## 工作台页签

### 事件计划

展示 EventUnit 顺序、标题、摘要、参与方、证据引用和时长建议。审核操作固定在面板底部，包括批准、提交修改和拒绝。

### 字幕脚本

按时间顺序展示 NarrationBeat：

- 字幕正文；
- 重要度；
- 对应 EventUnit 和证据；
- 关注实体；
- 目标时长；
- 编辑状态和 fingerprint 状态。

字幕修改会明确提示其下游蓝图和运行时将失效。主操作是“按字幕生成场景”。

### 场景蓝图

按字幕或 Scene Beat 展示主要演员、动作、空间关系、媒体辅助、镜头意图和降级策略。它回答“这一句字幕将如何被画面表达”，不直接暴露底层轨道 JSON。

### 资源

按 Beat 聚合模型、航迹、图片、视频和 GeoJSON。每项显示 available、fallback、missing、invalid 或 ambiguous 状态，并提供具体诊断。资源面板不把媒体列表与字幕上下文割裂。

### 参数

只提供运行时允许编辑的高价值参数，例如片段时长、媒体布局、音量、相机跟随方式和模型显示状态。字段由协议元数据或前端白名单驱动，未知字段保持只读。

### 预览

嵌入现有 Scene 播放器，提供播放、暂停、时间轴定位和重新生成入口。预览只有在 SceneProjectConfig 校验通过后启用。

## 模型配置

模型配置使用顶栏对话框或抽屉，不占用场景工作台页签。

提供商预设包括：

- DeepSeek；
- OpenAI；
- Qwen / DashScope；
- Kimi / Moonshot；
- Zhipu；
- OpenRouter；
- SiliconFlow；
- Ollama；
- LM Studio；
- vLLM；
- 自定义 OpenAI-compatible 服务。

字段包括提供商、Base URL、模型 ID 和密码输入框形式的 API key。支持发现模型、测试连接、保存、清除和恢复未保存值。

约束如下：

- 远程提供商只允许 HTTPS；
- 本地模式允许 loopback HTTP；
- API key 只提交给 Agent 的内存配置接口；
- API 响应不得返回 key；
- 页面刷新后可以查询“是否已配置”，但不能取回 key；
- Agent 重启后配置清空；
- 新 Turn 使用当前配置，运行中的 Turn 保留启动时的适配器快照。

## 前端组件边界

当前 `newScript/index.tsx` 只保留页面编排，不继续承载全部业务逻辑。建议拆分为：

- `NewScriptHeader`：标题、模型状态、导出、保存、预览和旧版入口；
- `ConversationPane`：消息列表、Agent Turn、审核卡片和滚动管理；
- `ChatComposer`：文本、附件、键盘操作和显式发送；
- `AgentTurn`：活动状态、失败状态和最终答复；
- `SceneWorkspace`：展开、折叠、调整宽度、页签和阶段路由；
- `EventPlanWorkspace`：EventPlan 检查与审核；
- `NarrationWorkspace`：字幕查看与编辑；
- `BlueprintWorkspace`：按 Beat 检查场景表达；
- `AssetWorkspace`：资源匹配与诊断；
- `RuntimeParamsWorkspace`：受控参数编辑；
- `PreviewWorkspace`：播放器承载；
- `ModelConfigDialog`：模型配置、发现和测试；
- `useWorkspaceStage`：把产物集合、审核状态和运行状态映射为 UI 阶段。

`NarrativePanel`、`ResourcePanel`、`ParamsPanel` 和 `EventPlanReview` 中仍有价值的展示逻辑可以迁入新边界，但不保留当前多种装饰性布局和模拟加载行为。

## 产物兼容层

UI 不直接用 `artifact.type.includes(...)` 决定业务流程。新增集中式阶段映射：

```ts
type WorkspaceStage =
  | 'conversation'
  | 'event-plan-review'
  | 'narration'
  | 'blueprint'
  | 'resolution'
  | 'runtime'
  | 'failed';
```

映射层负责：

- 识别 v1 EventPlan、NarrativePlan 和 CanonicalRuntimePlan；
- 识别 v2 NarrationPlan、SceneBlueprint、ResolvedScenePlan 和 SceneProjectConfig；
- 选择每类最新且未 superseded 的产物；
- 根据审核、运行和诊断状态确定默认页签；
- 对暂未支持的产物显示只读摘要，而不是让页面崩溃。

这样可以先完成页面外壳，同时等待主任务稳定新契约。

## 状态与错误处理

- API 请求失败时，在触发操作附近显示错误，不只显示全局 toast；
- Agent 失败保留完整 Turn，并展开失败活动；
- 工作台显示具体诊断，例如缺失的资源 ID、航迹边界或候选冲突；
- 可以恢复的失败提供“重试当前阶段”；需要用户修改的失败定位到对应字幕、蓝图或资源项；
- 失败不会把已批准 EventPlan 或已保存字幕回滚为旧版本；
- 产物 lineage 或 fingerprint 不匹配时，旧下游产物标记为过期并禁止预览；
- 无权限、未配置模型和模型连接失败分别显示，不合并为通用“请求失败”。

## 与主任务的并行边界

可以立即实现：

- 页面布局与自适应工作台；
- 顶栏和返回旧版；
- 对话区与显式附件发送；
- Agent Turn 成功、失败和审核状态；
- 模型配置 UI 外壳及前端状态；
- `WorkspaceStage` 映射框架和 v1 兼容；
- 现有 EventPlan、资源、参数和播放器的重新承载。

等待主任务契约后接入：

- NarrationPlan 编辑字段；
- SceneBlueprint 和 ResolvedScenePlan 的详细视图；
- SceneProjectConfig v2 新轨道参数；
- 字幕 fingerprint 导致的局部失效提示；
- 新资源解析和导演诊断。

UI 分支合并主任务提交时只补充映射和视图适配，不重写页面布局。

## 测试与验收

组件测试覆盖：

- 无产物时工作台隐藏且聊天区全宽；
- EventPlan 出现时工作台自动打开；
- 不同产物与审核状态映射到正确页签；
- failed Turn 显示“执行失败”并默认展开；
- 选择附件不产生网络请求；
- 纯文本、纯附件和文本加附件均可发送；
- 发送失败保留输入与附件，成功后清空；
- 模型配置永不把 API key 写入浏览器存储或显示在响应状态中；
- 未校验运行时禁止预览，校验通过后启用；
- 返回旧版保留 `projectId`。

桌面端 Playwright 验收覆盖：

1. 纯文本启动对话；
2. DOCX 作为待发送附件显示并显式发送；
3. EventPlan 审核后进入字幕阶段；
4. 场景工作台折叠、展开和调整宽度；
5. 模型配置打开、校验和清除；
6. 失败 Turn 与具体诊断显示；
7. SceneProjectConfig 就绪后打开现有播放器。

验收浏览器仅使用 desktop-chromium，不新增移动端截图或 Playwright 项目。

## 完成标准

- 页面不再显示空白“解析结果”区域；
- 用户可以通过纯文本或 DOCX 发起相同的场景生成流程；
- 附件在显式发送前不产生请求；
- 对话过程、审核状态、失败状态和最终答复语义准确；
- EventPlan、字幕、蓝图、资源、参数和预览具有清晰阶段归属；
- 模型配置支持主流云端和本地 OpenAI-compatible 服务且不持久化 key；
- UI 可兼容当前 v1 产物，并能通过集中映射接入主任务的 v2 产物；
- 新版默认可用，旧版入口继续有效。
